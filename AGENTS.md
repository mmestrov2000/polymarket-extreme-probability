# Repository Rules

This file defines global rules for all contributors and tools.

## Source of Truth
- `PROJECT_SPEC.md`
- `ARCHITECTURE.md`
- `TASKS.md`

## Branching and PR Rules
- Never commit directly to `main`.
- Use one branch per scoped task.
- Open a PR for all change-making work.
- Keep PRs small and reviewable.

## Quality Rules
- Add or update tests for behavior changes.
- Keep `TASKS.md` status in sync with delivery progress.
- Keep implementation aligned with `PROJECT_SPEC.md` and `ARCHITECTURE.md`.

## Team Collaboration Rules
- Follow tool-agnostic workflow in `docs/PLAYBOOK.md`.
- Codex skills and local scripts are optional accelerators, not team requirements.
